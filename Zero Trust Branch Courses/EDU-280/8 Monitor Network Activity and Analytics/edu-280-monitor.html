Analytics dashboards, traffic flow charts, interface health and link scoring, and packet and flow logs for session-level troubleshooting.

See the traffic, score the links, read the logs — visibility is the first line of defense.

Zero Trust Branch gives administrators rich, layered visibility — from high-level dashboard summaries of device counts and threat incidents, to per-session packet logs showing exactly who communicated with whom on which port. Dashboard visibility and traffic flow visualization

The ZTB Dashboard

The Zero Trust Branch dashboard gives administrators a high-level, real-time overview of the entire branch network. It's your launchpad for visibility, insights, and action.

PANEL 1
Insights Panel
Snapshot of total endpoints, network count, isolation incidents, and active threats. Click Learn more for deeper reports.
PANEL 2
Device Insights
Understand connected devices by brand and model, including IoT devices like IP cameras — broken down by network.
PANEL 3
Network Topology
Visualize connections between locations and the Internet using interactive charts. Use zoom and search tools to explore any node.

Traffic Flow Charts

The Traffic Flow Chart is one of the most powerful tools for understanding lateral movement inside your network. It helps you visualize how devices are communicating with each other — so you can spot risky patterns, validate policies, or simply see what's happening in real time.

  • Every device becomes a node in the graph
  • Every communication path becomes an edge connecting those nodes
  • Click any node to get more details — network, subnet, VLAN tag, and network type
  • Built on a graph database, giving you a dynamic, searchable view of all internal traffic flows
i
UPDATE FREQUENCY
The Traffic Flow Chart is not instantaneous — it may take up to 10 minutes after a communication event for it to show up. For real-time session visibility, use Flow Logs instead.

INTERFACE HEALTH & LINK SCORING

Measuring WAN link quality with Link Scoring

After defining segmentation policies, it's important to ensure reliable communication between devices. Interface health monitoring helps detect link failures early, enabling intelligent routing decisions.

What Is Interface Monitoring?

Link failures can cause significant disruptions in your network. Interface health monitoring works by sending continuous probes to a defined destination to track whether a link is up or down. An intelligent ZTB gateway detects these failures and adjusts routing decisions accordingly.

Link Scoring (0–30)

Zero Trust Branch supports Link Scoring on a scale of 0–30 based on three metrics: jitter, latency, and packet loss. Because these metrics have different units, the results are normalized over a window of 30 iterations to produce a consistent, comparable score.

0 — Poor102030 — Excellent
Latency (ms)
Measured delay across the link. High latency indicates congestion or a long physical distance between endpoints.
Jitter (ms)
Variation in delay. High jitter degrades real-time applications like VoIP and video — even if average latency is acceptable.
Packet Loss (%)
Packets observed as lost on the link. Any packet loss degrades TCP and UDP application performance significantly.
Normalized Score
Because jitter, latency, and packet loss use different units, the score is normalized over 30 iterations to give a single 0–30 health indicator.
Interface Statistics path: Infrastructure › Connectors › Edge › Zero Trust Branch › Site-Name › Sites › Interfaces
Click the score for the required WAN interface to open Interface Statistics 24h.

What Interface Statistics Show

  • Interface / Gateway / Type / MAC — identifies which interface and gateway the statistics belong to
  • Score — Link Scoring 0–30, representing overall link quality
  • Loss (%) — packet loss observed on the link
  • Latency (ms) — measured delay across the link
  • Jitter (ms) — variation in delay
✓
TROUBLESHOOTING TIP
Use the AVG Score, AVG Loss, and AVG Latency charts to confirm whether issues are persistent or intermittent over the last 24 hours. Persistent degradation suggests a link problem; intermittent spikes suggest congestion or a routing issue.
In Zero Trust Branch, what is the maximum Link Score a WAN interface can receive, and which three metrics are used to calculate it?
✕
Maximum score of 100, calculated using bandwidth, latency, and uptime
✓
Maximum score of 30, calculated using jitter, latency, and packet loss — normalized over a window of 30 iterations
✕
Maximum score of 30, calculated using bandwidth, throughput, and packet loss
✕
Maximum score of 10, calculated using jitter, latency, and uptime

MONITORING & LOGS

Packet logs and flow logs for session-level investigation

Monitoring tools in Zero Trust Branch help you troubleshoot issues, spot patterns, and keep track of what's happening inside your network — giving you just the right level of visibility, whether you're looking for big-picture insights or fine-grained details.

Packet Logs

Packet logs give you detailed visibility into the traffic flowing through your network. They help you analyze patterns and troubleshoot any issues by showing raw data for each communication event.

Path: Logs › Packet Logs

Packet Log Filters

Drill down into logs using a variety of filter criteria:

Source: IP address
Source: Port
Source: OS
Source: Browser
Source: Geolocation
Destination: IP address
Destination: Name
Destination: Country
Destination: Port
Host: Hostname
Host: OS

Flow Logs

Flow logs give you a summary of session-level communications between endpoints. Each flow is defined by a 4-tuple: source IP, destination IP, destination port, and protocol. The associated action and policy are also shown.

4-Tuple Per Flow
Source IP · Destination IP · Destination Port · Protocol. Shows the associated action (allow/deny) and policy.
Deduplication by Default
By default, only one entry is logged per hour for a given 4-tuple. For example, if Host A sends multiple DNS requests to 8.8.8.8, only one is logged in that hour.
i
INCREASING LOG FREQUENCY
Want more detail? You can disable throttling when creating or editing a policy to see more frequent flow log entries for specific traffic.
