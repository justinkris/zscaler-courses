Introduce Zero Trust Branch Concepts

The new branch reality, why legacy architectures fail, and how Zscaler Zero Trust Branch eliminates the attack surface at the edge.

The branch is the new perimeter — and Zero Trust is the answer.

As apps moved to the cloud and users scattered everywhere, the traditional hub-and-spoke WAN became a liability. Zero Trust Branch extends cloud-delivered security all the way to the branch edge — no appliances, no complexity, no trust by location.

NEW REALITY

Work from anywhere, apps everywhere

Hybrid work and cloud transformation have fundamentally disrupted traditional network and security models. Private applications have moved from data centres to the cloud, users access those apps over the public internet from any location, and organisations are deploying IoT and OT devices across branches, factories, and data centres at scale.

Core challenge: Legacy WAN solutions were built for a world where apps lived in data centres and users sat at fixed desks. That world no longer exists — and traditional architectures create complexity, security gaps, and performance problems that Zero Trust Branch is designed to solve.

One Size Does Not Fit All

Modern branches come in many shapes and sizes. Each has different connectivity and security requirements:

SMALL BRANCH
Users Only
Simple connectivity needs. A few desks, basic internet access. Minimal IoT exposure but still requires consistent Zero Trust policy enforcement.
MEDIUM BRANCH / STORE
Users + IoT Devices
Printers, cameras, POS terminals, card readers. Requires better user experience and consistent segmentation across managed and unmanaged devices.
LARGE CAMPUS / FACTORY
Users, IoT + OT Systems
Full stack: users, IoT, servers, OT systems. Comprehensive security and segmentation needed. Flat networks here are the highest-risk environment.

What Changed — and What to Prioritise

  • Apps moved to cloud — users access SaaS and private apps over the public internet, not through the data centre
  • Firewalls in legacy WAN create complexity and leave security gaps for cloud-destined traffic
  • Prioritise cloud and mobile-first access, risk reduction, and adaptability to new technologies
  • IoT and OT devices improve operations but dramatically expand the attack surface across many locations

LEGACY PROBLEMS

Five critical issues with legacy architecture

Traditional branch security relied on perimeter-based firewalls and hub-and-spoke MPLS — backhauling all traffic to the data centre before routing to the internet. As organisations virtualised the same on-premises firewall into cloud environments, the complexity multiplied. Attempting to replicate this model across multiple availability zones and cloud providers creates five distinct problems:

1
Expanded Attack Surface
More firewalls means more chances for missed patches, outdated software, and policy gaps. Each unpatched firewall becomes an open door for attackers. More devices to manage, more paths for attack.
2
Complex Connectivity
SD-WAN, IPSec tunnels, and VPN connections linking clouds and data centres require specialised expertise and constant monitoring. More tunnels = more complexity, more points of failure.
3
Lateral Movement Becomes Easier
Flat, connected networks allow attackers to move from one compromised workload to others. Traditional firewalls protecting the perimeter provide little resistance to east-west movement once inside.
4
High Cost and Policy Sprawl
Licensing multiplies across every appliance. Maintaining consistent policies across environments becomes near-impossible. Inconsistent policies create exploitable gaps for attackers.
5
Performance Drops with Advanced Features
Enabling SSL inspection and DLP on physical appliances reduces throughput significantly. Security comes at the direct cost of performance — forcing organisations to choose between security and usability.
!
KEY TAKEAWAY
More devices to manage, more paths for attackers, more spend for less throughput. Legacy architecture was not built for cloud. Trying to extend it creates compounding risk, cost, and complexity.

The Challenge by Traffic Type

  • Cloud Applications: Backhauling SaaS and cloud traffic to a regional data centre for inspection degrades performance — users experience high latency accessing apps that should route direct-to-cloud
  • Hybrid Workforce: Home users often have better connection consistency than office users because branch traffic is hairpinned through congested WAN links
  • IoT and OT Devices: Connected devices improve operations but can't run agents — they require network-level security that legacy perimeters cannot consistently provide across many locations

BRANCH EVOLUTION

Four stages of branch network security

Branch network security has evolved through four distinct stages. Each stage solved the problems of its era but introduced new limitations. Zero Trust Branch is the culmination of this evolution — solving the fundamental flaws of every stage before it.

STAGE 1
Secure Router
MPLS or basic internet connections to data centre. Simple router ACLs, maybe stateful inspection. All traffic backhauled for inspection. WAN termination with inbound ACLs only.
Challenge: Insecure by design with poor performance. ACLs provide minimal protection; backhauling creates latency that degrades user experience.
STAGE 2
Firewalls at Branch
Dedicated NGFW added for branch protection. Faster internet links but still backhauling to data centre. Stateful firewalls with on-box security stack.
Challenge: Costly hardware procurement and complex appliance management. Traffic hairpin to data centre continues to degrade performance for branch users.
STAGE 3
SD-WAN
Intelligent path selection across multiple links. Direct internet access (DIA) breakouts at branch. Broadband as WAN replacement with centralised management.
Challenge: Required full security stack at every branch — the "fat branch" problem. Massive appliance sprawl with high CapEx/OpEx and no consistent policy enforcement.
STAGE 4
Zscaler Zero Trust Branch
Any internet connection including commodity broadband — no MPLS needed. Cloud-delivered Zero Trust replaces the entire branch security stack. Direct-to-cloud inspection via Zscaler's global security cloud.
Outcome: Simple, secure, and cost-effective with zero branch appliances. Consistent Zero Trust policy enforcement everywhere users and devices connect.
★
EXAM POINT
Zero Trust Branch is the most effective security strategy against lateral threats. It hides private apps behind the Zero Trust Exchange, auto-provisions endpoints with /32 subnet masks for isolation, and ensures all device communication goes through the Zero Trust gateway — preventing lateral movement entirely.

Branch Connector — the engine of Zero Trust Branch

The Branch Connector is the only appliance in the Zscaler family designed specifically for branch-and-device segmentation. It is deployed directly at the branch and acts as the enforcement point for all local network traffic — including traditional endpoints, IoT sensors, and OT systems that cannot run software agents.

  • Deploy: Plug-and-play — standard gigabit Ethernet ports, AC power supply
  • Connect: Forms GRE tunnel to Zero Trust Exchange; branch traffic routed for ZIA, ZPA, and guest Wi-Fi
  • Secure: TPM 2.0 chip provides Zero Trust Provisioning authentication credentials to Zscaler cloud, then downloads provisioning template
!
M&A USE CASE
During mergers and acquisitions, Branch Connector enables quick access to internal applications without agent installation or network reconfiguration. It can also act as an App Connector, providing secure remote access to local systems — making it a powerful tool during transitions.

ZERO TRUST SASE

Zscaler Zero Trust SASE solution

Secure Access Service Edge (SASE) is the convergence of networking and security into a unified cloud-delivered framework. While some SASE solutions only address user-to-app access, Zscaler's Zero Trust SASE extends Zero Trust principles to branches, factories, and data centres — eliminating the need for traditional security and networking appliances.

The problem with traditional SD-WAN SASE: SD-WAN architectures expand the attack surface and allow lateral threat movement. Zscaler Zero Trust SASE — built on Zero Trust Branch — reduces business risk and network complexity by removing appliances and enforcing policy in the cloud.

Four Advantages of Zscaler Zero Trust SASE

1
Zero Trust Beyond Users
Enforce Zero Trust security for IoT and OT devices, servers, and guests in branches — not just managed user devices. Every connection type is covered.
2
Unified Security and Network Access
Single-vendor SASE framework with a simpler, proxy-based architecture. One platform for internet, SaaS, and private app security — no appliance sprawl.
3
Consistent Business Policy
Reduce risk and complexity by consolidating security policies and administration. One policy engine for all users, devices, and locations — no policy drift or gaps.
4
Digital Experience Monitoring
ZDX provides end-to-end visibility into performance across the entire path from branch device to cloud app. Resolve issues quickly before they impact productivity.

SSE vs Zero Trust Branch

  • SSE (Secure Service Edge) — cyber threat protection, cloud/SaaS/endpoint security, DLP, user-risk assessment. Covers users to apps.
  • Zero Trust Branch — extends Zero Trust architecture to branches, factories, and data centres. Covers IoT, OT, servers, and guests that SSE alone cannot protect.
  • Together, SSE + Zero Trust Branch = Zero Trust SASE — the complete solution for every user, device, and location.
