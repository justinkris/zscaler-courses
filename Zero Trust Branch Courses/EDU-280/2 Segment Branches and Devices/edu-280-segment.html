Segment Branches and Devices

Device segmentation using /32 subnet masks, hardware appliance sizing, and use cases for east-west firewall replacement, IoT/OT isolation, and M&A acceleration.

Every device its own island — that's the power of a segment of one.

Traditional VLANs trust devices on the same segment. Zero Trust Branch gives every device its own /32 network identity, forces all traffic through the gateway, and enforces policy before anything talks to anything else — eliminating lateral movement at the source.

SEGMENTATION BASICS

Zero Trust segmentation fundamentals

Zscaler Zero Trust Branch applies segmentation at four distinct layers. Each layer restricts communication to what is explicitly authorised, removing the implicit trust that makes flat networks so dangerous.

1
User Segmentation
Restrict user access to only the resources essential for their role. No user can reach anything not explicitly permitted — regardless of which network segment they sit on.
2
Workload Segmentation
Prevent cross-application threats in data centres and clouds. Workloads are isolated from each other — a compromised app cannot pivot to adjacent systems.
3
Branch / Campus Zones
Secure branches and campuses through isolated zones. Users, IoT, and OT are grouped by function and restricted from communicating outside their authorised zone.
4
Device Segmentation
Apply granular security policies to isolate individual devices — even IoT and OT that cannot run agents. Each device gets its own isolated identity, reducing internal risks to near zero.

The "Segment of One" Concept

Core principle: Zero Trust Branch assigns a /32 subnet mask to each device — making every device its own network. This forces all traffic to route through the Zero Trust gateway, where segmentation policies restrict any unauthorised east-west communication. No device can talk to another without explicit policy permission.

Think of it like an office building with a smart access control system. Each employee (device) gets a personalised badge (/32 identity) that only opens the doors they're authorised to use. Even if an attacker steals one badge, they can't access any other floor. The Zero Trust gateway is the security desk that validates every movement.

What Zero Trust Branch Eliminates

  • Static firewall rules and manual ACL configurations
  • East-west firewall appliances at branch locations
  • Dependency on NAC (Network Access Control) appliances
  • Agent-based segmentation methods
  • Risk of lateral threat propagation inside the network
  • Operational overhead for managing device and policy changes

HOW IT WORKS

The 5-step Zero Trust Branch security framework

Zero Trust Branch operates as the default gateway for all branch traffic. Every device must pass through it before communicating with any other device. The framework executes five steps automatically — no manual configuration required for day-to-day operations.

1
Act as the Security Desk
Zero Trust Branch becomes the default gateway for the branch — like the main security desk at the entrance of each floor. Every device must pass through it before talking to another device. No direct device-to-device communication is permitted.
2
Assign Unique Badges (/32 Identity)
Each device receives its own isolated network identity via a /32 subnet mask — like a personalised access badge. This uniquely identifies every device and forces all its traffic through the gateway for policy evaluation.
3
Sort by Department (Auto-Classify)
The system automatically classifies devices in real time into groups: IT, IoT, OT, guests. Like employees grouped into departments. Group-based policies are applied automatically — no manual VLAN configuration or ACL updates needed.
4
Enforce Access Rules
Access rules are enforced per group — like only certain employees being able to open certain doors. Example: only authorised admins can access camera feeds. IoT devices cannot reach corporate servers. OT systems are isolated from user networks.
5
Emergency Response — Ransomware Kill Switch
If a threat is detected (such as ransomware activity), the system triggers an emergency stop — instantly isolating risky traffic across the entire branch. The Ransomware Kill Switch contains a threat before it can spread laterally to other devices.

HARDWARE & VIRTUAL APPLIANCES

Sizing your Zero Trust Branch deployment

Zero Trust Branch is available as four hardware appliances and two virtual appliance configurations. The right choice depends on branch size, endpoint count, and throughput requirements. All are deployed in Gateway Mode — inline as the branch gateway, enabling secure direct access to cloud applications and the internet over one or more ISP connections.

GATEWAY MODE
All appliances deploy in Gateway Mode (Inline) — the appliance serves as a local router for branch devices while an existing WAN device handles internet connectivity. It intelligently selects the best link quality across dual ISP connections and provides seamless failover for high availability.

INTERNAL USE CASES

Securing internal branch communication

Zero Trust Branch addresses three core internal security scenarios — each replacing a legacy approach with dynamic, policy-driven segmentation that requires no manual ACLs, no static VLANs, and no agent installation.

USE CASE 1
East-West Firewall Replacement
A manufacturing facility with OT systems, patient monitors, and lab equipment all connected to the corporate network. By deploying Zero Trust Branch, the facility automatically groups these devices, isolates each into its own segment of one, and enforces policy — no downtime, no hardware changes.
Eliminates static firewall appliances and fixed VLAN security
Dynamic segmentation adapts automatically to network changes
No manual ACLs or rule updates needed
Easier to scale and support as the network grows
USE CASE 2
IoT / OT Segmentation
A smart factory needs to isolate IoT/OT devices without disrupting operations. Zero Trust Branch automatically groups these devices, identifies access boundaries around critical systems, and enforces segmentation — no downtime, no disruptions, no NAC appliances required.
Automatic IoT/OT device discovery and classification
Each OT system isolated into its own segment of one
Restrict factory floor access to known MAC addresses
Autonomous grouping eliminates need for manual VLAN management
USE CASE 3
Automatic Device Discovery and Classification
A factory struggles to track all connected devices — sensors, barcode scanners, and IoT devices go undetected and unmonitored. Zero Trust Branch automatically identifies and classifies every device in real time, creates accurate inventory, and detects anomalies through traffic flow analysis.
Automatic network asset discovery and classification
Real-time automapping and policy management
Device-to-device and user-to-device traffic mapping
Reduces blind spots in IoT/OT/IT asset visibility
Enables precise segmentation policies based on real device behaviour

EXTERNAL USE CASES

Securing external branch connectivity

Zero Trust Branch also handles three key external connectivity scenarios — replacing legacy site-to-site VPNs, accelerating M&A integrations, and providing AI-powered IoT visibility across distributed locations.

USE CASE 4
Replace Site-to-Site VPN
Connect branches directly to private applications in data centres or public clouds — without extending the corporate network or exposing VPN infrastructure. Applications are hidden from discovery behind the Zero Trust Exchange. Access is restricted to named, verified entities only — identity, context, and policy adherence are verified before any connection is allowed.
Direct, secure branch-to-app connections without VPN tunnels
Zero Trust network overlay prevents lateral movement
Private apps are hidden — not discoverable from the internet
Eliminates the attack surface that VPN gateways create
USE CASE 5
Accelerate M&A Integration
Company A is integrating with Company B, which uses an identical IP address scheme. By deploying Zero Trust Branch, the acquiring company connects and provisions applications across both networks without IP address changes, overlaps, or lateral threat movement. Organisation-based policies control access across sites from Day 1.
No IP address changes required — works even with overlapping schemes
Onboard new sites in days, not months
Operational simplicity — no network reconfiguration needed
Successful Day 1 operations with full policy enforcement
USE CASE 6
AI-Powered IoT Discovery & Classification
Teams face blind spots when unsanctioned, undiscoverable devices connect to branch office networks — increasing vulnerability and expanding the attack surface. Zscaler identifies and classifies devices using AI to give IT teams deeper visibility into behaviour, enabling accurate access control policies.
Create non-routable IoT islands — devices cannot reach outside their zone
AI-powered IoT Discovery Report with full device classification
Deeper visibility and control over unmanaged device behaviour
Policies automatically adapt as new devices are discovered
