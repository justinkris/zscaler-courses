WHAT IS INCIDENT MANAGEMENT?

Identifying, analysing, and resolving incidents

Incident Management is a process of identifying, analysing, and resolving critical incidents which could lead to issues in an organisation. It is a critical component of any security framework — helping organisations mitigate the impact of security incidents, minimise downtime, and prevent future occurrences.

Incidents can involve:

  • Security threats (e.g., malware, phishing attempts)
  • Network or application failures
  • Unusual or suspicious activity flagged for investigation
In data security context: When adopting DLP and CASB (SaaS Security API) capabilities, alerts are generated that require administrator teams to investigate and respond. Zscaler provides tools to manage these alerts through multiple channels — delegating violations to end users where appropriate.

NOTIFICATION METHODS

How Zscaler notifies users and admins

The first priority in incident management is enabling end users to self-serve on violations — delegating responsibility and reducing admin burden. Zscaler provides several notification channels that can be customised with your organisation's logo and messaging.

End-User Notification Options

🌐
Browser-Based Notification
Customisable page shown when a user attempts a blocked action. Can block or allow the transaction while notifying the user. Supports form POST for justification feedback back to the admin.
💬
Slack / Teams Connector
Notifies users via Slack or Microsoft Teams when a violation is detected. Users can provide justification directly within the chat platform — many organisations prefer this over browser notifications.
🖥️
Zscaler Client Connector Pop-Up
A pop-up on the endpoint via the Zscaler Client Connector — even for endpoint DLP violations. Same idea: block or allow, ask for justification, or coach the user not to repeat the violation.

Administrator Notification and Integration Options

  • Email notifications — SMTP-based alerts sent to DLP admins when incidents are triggered
  • Secure ICAP protocol — for integration with on-premises systems
  • SIEM integration — real-time log streaming to Splunk, QRadar, and other SIEM tools
  • ServiceNow integration — automated ticket creation for incident tracking
  • On-premises incident receiver — for archiving incidents locally
★
KEY ANSWER
When asked how Zscaler notifies users about policy violations: the answer is browser-based notifications, Slack/Teams connectors, or the Zscaler Client Connector — NOT by sending personal emails, shutting down systems, or requiring calls to IT support.

ZSCALER WORKFLOW AUTOMATION

Automated, closed-loop incident management

Zscaler Workflow Automation (WFA) streamlines data security workflows by automating incident management — enabling quick remediation of data protection incidents. It automatically groups incidents based on various attributes and provides a complete audit trail.

WFA integrates with:

  • Zscaler Internet Access (ZIA) — captures Data Protection incidents generated from DLP policies
  • Zscaler Digital Experience (ZDX) — generates enriched tickets for different events triggered by data defined in ZIA
  • Business Insights — manages usage of SaaS application licences provisioned to end users

Key Features

Incident Grouping
Automatically groups incidents into prioritised incident groups based on the originating user, the user's manager, and the triggering activity or policy. Groups can be assigned to different admins for parallel handling.
Workflow Management Tools
Notifies users for incident justification or escalates incidents to managers for approval. Admins can view the triggering content, apply remediation notes, and track the full lifecycle of each incident.
Automated Workflows
Manages and remediates DLP incidents efficiently — escalating violations to relevant personnel, automating ticket creation, and resolving incidents with a full audit trail of every action taken.

Incidents Page

Workflow Automation Incidents page — All: 502, Open: 26, Unassigned: 488, Resolved: 68, Waiting Feedback: 408, Escalated: 51

Incident Analytics

Incident Analytics — Time to Triage/Resolve, Incident by Status, False Positive Incidents by Rule and Dictionary. Helps identify patterns and improve DLP policy accuracy.
✓
INCIDENT LIFECYCLE
1. Incident triggered → admin reviews policy and triggering content.
2. Admin notifies end user (email/Slack/browser) requesting justification.
3. User submits justification → admin reviews feedback.
4. If escalated → manager/approver receives escalation with higher detail.
5. Manager can accept, escalate further, or deny.
6. Resolution recorded → count updated → incident closed.
